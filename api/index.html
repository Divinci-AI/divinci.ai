Divinci API

Integrate powerful AI capabilities into your applications with our comprehensive API. Build custom AI solutions that leverage RAG systems, vector embeddings, and document processing.

Get API Keys View Documentation

API Reference

The Divinci AI API provides REST endpoints for managing workspaces, knowledge bases, releases, and more. All requests require authentication via API key or OAuth.

Getting Started

Base URL: https://api.divinci.app/api/v1. Authenticate with your API key using the Authorization: Bearer divinci_your_key header or x-api-key header. Generate API keys from your workspace dashboard.

Install the CLI for quick access: npm install -g @divinci-ai/cli, or use the Server SDK: npm install @divinci-ai/server.

API v1 Endpoints

Authentication

  • POST /api/v1/auth/validate-key
    Validate an API key and return its permissions, workspace ID, and billing model

Workspaces

  • POST /api/v1/workspaces
    Create a new workspace (requires OAuth login)
  • GET /api/v1/workspaces
    List your workspaces
  • GET /api/v1/workspaces/{id}
    Get workspace details
  • PATCH /api/v1/workspaces/{id}
    Update workspace title or description
  • DELETE /api/v1/workspaces/{id}
    Delete a workspace and all associated resources

Releases

  • GET /api/v1/releases
    List all releases for your workspace
  • POST /api/v1/releases
    Create a new release draft with model, RAG, and MCP configuration
  • GET /api/v1/releases/{id}
    Get release details
  • PATCH /api/v1/releases/{id}
    Update a release draft (model, tools, RAG index, MCP config)
  • POST /api/v1/releases/{id}/publish
    Publish a release draft to make it available
  • DELETE /api/v1/releases/{id}
    Delete a release draft

RAG (Knowledge Base)

  • POST /api/v1/rag/search
    Search your knowledge base with semantic + hybrid search
  • GET /api/v1/rag/files
    List RAG files and their processing status
  • POST /api/v1/rag/files
    Upload a document to your knowledge base (multipart/form-data — do not use application/json)
  • GET /api/v1/rag/targets
    List available RAG vector indexes

User Groups

  • POST /api/v1/groups
    Create a user group for team access management
  • GET /api/v1/groups
    List your groups
  • POST /api/v1/groups/{id}/invite
    Invite a user to a group by email
  • DELETE /api/v1/groups/{id}/members
    Remove a member from a group

Permissions

  • GET /api/v1/permissions
    Get permission configuration for your workspace
  • GET /api/v1/permissions/mine
    Get your own effective permissions
  • POST /api/v1/permissions/users
    Grant permissions to a user
  • POST /api/v1/permissions/groups
    Grant permissions to a user group

Transcripts & Chat

  • GET /api/v1/transcripts
    List chat transcripts
  • GET /api/v1/threads
    List chat threads

Additional Endpoints

  • GET /api/v1/config
    Get workspace configuration
  • GET /api/v1/audio-transcripts
    List audio transcripts
  • GET /api/v1/connectors
    List data connectors (Google Drive, Dropbox, S3, etc.)
  • GET /api/v1/fine-tune
    List fine-tuning jobs
  • POST /api/v1/tts
    Generate speech from text
  • GET /api/v1/export
    Export workspace data

# Validate your API key
curl -X POST https://api.divinci.app/api/v1/auth/validate-key \
  -H "Authorization: Bearer divinci_your_api_key"

# List releases
curl https://api.divinci.app/api/v1/releases \
  -H "Authorization: Bearer divinci_your_api_key"

# Upload a file to your knowledge base (multipart/form-data — do NOT set Content-Type manually)
curl -X POST https://api.divinci.app/api/v1/rag/files \
  -H "Authorization: Bearer divinci_your_api_key" \
  -F "file=@/path/to/document.txt;type=text/plain" \
  -F "ragVectorId=your-rag-vector-id" \
  -F "title=My Document"

# Search your knowledge base
curl -X POST https://api.divinci.app/api/v1/rag/search \
  -H "Authorization: Bearer divinci_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "query": "What is the return policy?",
    "limit": 5
}'

# Grant permissions to a team member
curl -X POST https://api.divinci.app/api/v1/permissions/users \
  -H "Authorization: Bearer divinci_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "email": "teammate@example.com",
    "permissions": ["whitelabel:rag", "whitelabel:config"]
}'

npm install -g @divinci-ai/cli

# Login with OAuth
divinci auth login

# Create a workspace
divinci workspace create --title "My AI App" --description "Custom AI assistant"

# Set your active workspace
divinci workspace use <workspace-id>

# Create a release from an MCP server
divinci release create-from-mcp \
  --url "https://your-mcp-server.com/api/v1/mcp" \
  --name "My AI Release"

# Upload a file to your knowledge base
divinci rag upload --file /path/to/document.txt --rag-vector-id <rag-vector-id> --title "My Document"

# Manage team access
divinci group create --name "Engineering" --slug "engineering"
divinci group invite <group-id> --email "dev@example.com"
divinci permissions grant --group engineering \
  --permissions "whitelabel:rag,whitelabel:config"
import { DivinciServer } from "@divinci-ai/server";

const divinci = new DivinciServer({
  apiKey: process.env.DIVINCI_API_KEY,
  defaultWorkspaceId: process.env.DIVINCI_WORKSPACE_ID,
});

// List releases
const releases = await divinci.releases.list({
  workspaceId: "your-workspace-id"
});

// Upload a file to your knowledge base (multipart/form-data — do not send application/json)
const formData = new FormData();
formData.append("file", new Blob([textContent], { type: "text/plain" }), "document.txt");
formData.append("ragVectorId", "your-rag-vector-id");
formData.append("title", "My Document");
await fetch(`https://api.divinci.app/api/v1/rag/files`, {
  method: "POST",
  headers: { Authorization: `Bearer ${process.env.DIVINCI_API_KEY}` },
  // Do NOT set Content-Type — let fetch set it automatically with the multipart boundary
  body: formData,
});

// Search knowledge base
const results = await divinci.rag.search(
  "workspace-id",
  "collection-id",
  "What is the return policy?"
);

// Manage groups and permissions
const { group } = await divinci.groups.create({
  name: "Engineering",
  slug: "engineering"
});
await divinci.groups.invite(group._id, "dev@example.com");
await divinci.permissions.grantToGroup(
  "engineering",
  ["whitelabel:rag", "whitelabel:config"]
);

SDK & CLI

Install our official packages from npm to integrate Divinci AI into your applications.

CLI

Manage workspaces, releases, RAG, groups, and permissions from the terminal. Supports OAuth login and API key auth.

@divinci-ai/cli

TypeScript-first server SDK for Node.js. Manage releases, RAG search, groups, permissions, and more with full type safety.

@divinci-ai/server
npm install @divinci-ai/server
npm GitHub

Connect Divinci to Claude, Cursor, or any MCP-compatible AI assistant. Exposes RAG search, chat, releases, and more as MCP tools.
